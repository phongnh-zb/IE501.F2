How to run the pipeline

Requires Hadoop, HBase, and Spark installed locally. Execute from the project root.

1
Full pipeline (recommended)
Starts services, ingests data, runs ETL, trains models, writes to HBase.
python3 main.py
2
Start services only
Starts Hadoop, HBase, and the Thrift server if not already running.
bash scripts/start_services.sh
3
Check service status
Prints the status of all infrastructure components.
bash scripts/status.sh
4
Start the web dashboard
Runs the Flask application in a separate terminal.
python3 webapp/app.py

The Docker container runs the web dashboard only. Hadoop, HBase, and the pipeline must still run on the host. Set HADOOP_HOST and HBASE_HOST to reach them.

Development foreground · hot reload
Mounts templates and static files from the host — changes apply without rebuilding.
docker compose up
Production detached
Bakes all files into the image and runs in the background.
docker compose -f docker-compose.yaml -f docker-compose.prod.yaml up -d
Rebuild image
Force a full rebuild after installing new dependencies or changing the Dockerfile.
docker compose up --build
